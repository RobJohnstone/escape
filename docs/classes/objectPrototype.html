objectPrototype Class

Defined in: client/js/objectPrototype.js:1
Module: objectPrototype

Extend Object.prototype with helper methods to simplify inheritance

Escape uses prototypal style inheritance rather than the constructor / pseudo-classical approach used by many javascript developers. The "blueprints" for each type of object are defined as objects in files named "Prototype.js". A new object of that type can be created using "var newObj = Prototype.create();". To create a new prototype based upon a previous one use "var newPrototype = Prototype.extend();"

It could be argued that extending Object.prototype in this way is an anti-pattern due to the effect it could have on code written by those who are unaware that it has been extended. However, the code below takes account of the most common types of conflict and employs suitable preventative measures so this sort of problem should not occur.
